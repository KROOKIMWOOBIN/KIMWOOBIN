## 왜 이런 구조를 사용할까?
- 기존 MVC 구조는 단순하고 빠르게 개발 가능하다. 하지만 프로젝트가 커질수록 이런 문제가 생긴다.
```
Controller -> Service -> Repository
```
### 문제
1. Service에 비즈니스 로직 집중
2. DB 코드와 비즈니스 로직 혼재
3. 외부 API 코드까지 Service에 침투
4. 테스트 어려움
5. 변경 영향 범위 증가

### 해결책
- 같은 역할 분리 구조
1. Application
2. Domain
3. Infrastructure
4. Interface

## 전체 구조
```
interfaces
 └─ api
     └─ controller

application
 └─ service

domain
 ├─ entity
 ├─ vo
 └─ service

infrastructure
 ├─ persistence
 ├─ redis
 ├─ kafka
 └─ external

support
 └─ error
```

### 각 계층 역할

#### application
- 도메인 객체를 조합하여 사용자의 요구사항(Usecase)을 처리합니다. 트랜잭션 관리와 흐름 제어를 담당합니다.
##### 사용 예시
```
1. 아이디 중복 검사
2. 비밀번호 암호화
3. 회원 생성
4. 저장
5. 이벤트 발행
```
##### 예시 코드
```
@Service
@RequiredArgsConstructor
@Transactional
public class UserRegisterService {

    private final UserRepository userRepository;
    private final PasswordEncoder passwordEncoder;

    public void register(RegisterCommand command) {

        if (userRepository.existsByUserId(command.userId())) {
            throw new DuplicateUserException();
        }

        String encodedPassword =
                passwordEncoder.encode(command.password());

        User user = User.create(
                command.userId(),
                encodedPassword,
                command.name(),
                command.email()
        );

        userRepository.save(user);
    }
}
```

#### domain
- 시스템의 핵심 비즈니스 로직과 상태를 담습니다. 엔티티(Entity), 값 객체(Value Object), 도메인 서비스가 위치합니다.
##### 사용 예시
```
비밀번호 정책 변경
결제 제한 정책 추가
주문 상태 변경 규칙 수정
```
##### Entity 
- 식별자(ID)를 기준으로 구분되는 객체
- 상태 변경 가능
- 생명주기를 가짐
- 같은 ID면 같은 객체로 판단

##### VO
- 값 자체가 의미인 객체
- 불변(Immutable) 권장
- 값이 같으면 같은 객체
- 생성 시 검증 수행


#### infrastructure
- DB 저장, 메시지 큐(Kafka, Redis) 송신, 외부 API 호출 등 기술적인 구현체를 담습니다.
##### 사용 예시
```
"비즈니스"가 아니라
"기술 구현"
```
##### 예시 코드
```
@Repository
@RequiredArgsConstructor
public class UserJpaRepository
        implements UserRepository {

    private final SpringDataUserRepository repository;

    @Override
    public void save(User user) {
        repository.save(UserEntity.from(user));
    }

    @Override
    public boolean existsByUserId(String userId) {
        return repository.existsByUserId(userId);
    }
}
```

#### interfaces.api
- 외부 사용자의 요청을 받는 접점입니다. Controller, DTO 변환 등이 여기에 해당합니다.

#### support.error
- 시스템 전반에서 발생하는 예외 처리 공통 로직, 에러 코드 정의 등을 관리합니다.
